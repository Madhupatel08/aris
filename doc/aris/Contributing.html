11 Contributing

Everyone no doubt wants to help out us weary developers with our work, so here’s how you do it:

  1. Check out the latest source from the git repo: git clone https://git.savannah.gnu.org/git/aris.git
  2. Hack away and make your changes. Commit them as you go, following the coding standards of the GNU Project
  3. Format patches for your changes using git format-patch.
  4. Tar up the patches with tar czf aris-patches.tar.gz *.patch. (We don’t have a preference for compression algorithm; we just use gzip here as an example.)
  5. Email the tarball of patches to bug-aris@gnu.org.

If you just want to submit bug reports, that’s fine, too. Submit the report to bug-aris@gnu.org, and try to be as specific as possible.
